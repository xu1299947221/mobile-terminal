# 手机浏览器和 PWA 使用说明

本文记录手机浏览器全屏、添加到主屏幕和常见问题。这里的 PWA 只是让网页更像 App 打开，不影响服务器端 `tmux`、`ttyd` 和登录权限。

## 推荐使用方式

### iOS / iPadOS

推荐使用 Safari：

1. 用 Safari 打开公网地址。
2. 登录后进入项目页。
3. 点击 Safari 底部分享按钮。
4. 选择“添加到主屏幕”。
5. 从桌面图标重新打开。

iOS 上 Chrome、Edge 等浏览器底层仍使用 WebKit，添加到主屏幕的能力通常还是 Safari 路径最稳定。普通浏览器标签页里不能像桌面浏览器 `F11` 一样强制隐藏所有浏览器工具栏。

### Android

Android 推荐优先尝试：

1. Edge
2. Chrome
3. Samsung Internet
4. 其他支持“添加到主屏幕”的浏览器

如果浏览器内置全屏按钮可用，也可以直接使用浏览器全屏，不一定必须安装 PWA。

## Android 权限注意事项

国产 Android ROM 经常会拦截浏览器创建桌面入口。遇到“正在安装”“添加后没有图标”“点安装后跳到浏览器应用详情页”等情况时，先检查浏览器应用权限。

以 Edge 为例，实测需要给 Edge 开启“创建快捷方式”权限后，才能正常添加到手机桌面。

常见路径类似：

```text
系统设置
  -> 应用
  -> Edge / Chrome / 对应浏览器
  -> 权限 / 其他权限
  -> 创建快捷方式 / 桌面快捷方式 / 添加主屏幕快捷方式
  -> 允许
```

不同系统叫法不完全一样，可能出现在：

- 权限管理
- 其他权限
- 特殊应用权限
- 桌面快捷方式
- 创建桌面图标
- 添加主屏幕快捷方式

小米、OPPO、vivo、荣耀等系统尤其需要检查这类权限。

## Chrome 卡在“正在安装”

Chrome Android 安装 PWA 可能会走 WebAPK / Google 服务相关链路。国内网络、Google Play 服务不可用、系统限制后台安装时，可能出现：

- 一直显示“正在安装”。
- 第二次添加时仍提示正在添加第一次的页面。
- 没有任何权限弹框。
- 桌面没有生成图标。

这通常不是服务端代码问题，也不是网站登录权限问题。可以尝试：

1. 清除该站点数据。
2. 清除 Chrome 缓存。
3. 检查 Chrome 是否有“创建快捷方式”权限。
4. 确认系统桌面允许浏览器创建图标。
5. 换 Edge 或 Samsung Internet。

项目已经做了 Android PWA 兼容处理：

- `manifest.webmanifest` 使用 `display: standalone`。
- 保留 `display_override` 中的 `fullscreen`。
- 使用标准 8-bit PNG 图标。
- `manifest.webmanifest`、`sw.js`、`icon-*.png` 不缓存，避免手机拿旧版本。

如果 Chrome 仍卡住，优先认为是手机系统或网络环境导致。

## Edge 添加后跳到应用详情页

如果 Edge 点击“添加至手机”或“安装”后跳到 Edge 应用详情管理界面，通常表示系统要求给 Edge 补权限。

处理方式：

1. 在 Edge 应用详情页进入权限或其他权限。
2. 找到“创建快捷方式”或类似权限。
3. 设置为允许。
4. 回到网页重新添加。

实测开启这个权限后可以继续添加。

## QQ 浏览器等国内浏览器

QQ 浏览器、夸克、UC 等浏览器对标准 PWA 的支持不稳定。可能可以进入浏览器全屏，但不一定提供标准的“安装应用”或“添加到主屏幕”入口。

这类浏览器可以作为临时全屏使用方案，但不建议作为 PWA 安装的主方案。

## 清理旧安装状态

如果浏览器之前添加失败，建议清理旧状态后再试：

1. 删除桌面上已有的旧图标。
2. 系统应用列表里如果出现 `Terminal` 或 `mobile-terminal`，先卸载。
3. 浏览器里清除该站点数据。
4. 重新打开公网地址。
5. 再执行添加到主屏幕。

Android Chrome 的“正在安装”状态有时会被本地缓存，清站点数据不一定立刻清掉；这种情况下换 Edge 或其他浏览器更省时间。

## 和全屏按钮的区别

项目页里的“安装/全屏”按钮分两类能力：

- 支持 Fullscreen API 的浏览器：可以直接进入浏览器全屏。
- 支持 PWA 安装的浏览器：可以安装或添加到主屏幕。

浏览器全屏通常只对当前标签页有效。PWA / 添加到主屏幕是从桌面图标启动，体验更接近 App。

在当前项目里，`ttyd` 会通过 `/ttyd-view/:slug` 在应用内全屏容器打开，避免从全屏页跳到新浏览器页面。
