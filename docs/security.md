# 安全设计

说明：本文档正文使用中文；产品名、命令名、协议名、路径和代码标识保留英文原文，避免安全配置时产生歧义。

本文说明 `mobile-terminal` 的安全边界和必须遵守的部署规则。

## 基本原则

- 应用不提供公开注册。
- 所有用户必须由管理员创建。
- 管理员可以禁用用户、重置密码、调整角色和分配项目权限。
- 普通用户不能通过 URL 参数访问任意服务器路径。
- 所有项目路径只能来自管理员配置。

## 登录和会话

- 用户密码使用 `Argon2id` hash 存储。
- 登录态使用 `HTTP-only cookie`。
- HTTP 页面、API 和 WebSocket 长连接都必须校验登录态。
- 未登录用户不能访问项目、终端、后台接口。
- 正式环境必须使用强管理员密码。

## 用户角色

当前角色分为：

```text
admin   管理用户、项目、会话和所有终端访问
member  访问被授权项目，通常可写
viewer  只读访问被授权项目
```

项目权限分为：

```text
none   无权限
read   只读终端
write  可写终端
```

只读用户：

- 可以查看自研 xterm.js 终端输出。
- 不能向自研终端写入输入。
- 不能打开 ttyd 备用终端，因为 ttyd 是完整交互终端，难以可靠限制只读。

## 终端安全

- 每个项目绑定一个明确的 `tmux` 会话。
- 系统只能管理自己配置过的会话。
- 不得误杀与本系统无关的 `tmux` 会话。
- 手机断开连接不能杀掉底层任务。
- WebSocket 长连接每次连接都必须重新校验项目权限。
- 终端输入写入前必须再次校验用户是否有写权限。

## ttyd 备用终端

`ttyd` 只作为备用终端，不作为公网直接入口。

要求：

- `ttyd` 进程只能监听 `127.0.0.1`。
- 外部访问必须先经过 `mobile-terminal` 登录和项目权限校验。
- Cloudflare 隧道不能直接指向 `ttyd` 端口。
- 只读用户不能访问 `ttyd`。

## Cloudflare 隧道

公网入口使用：

```text
https://terminal.example.com -> Cloudflare 隧道 -> http://127.0.0.1:3020
```

安全要求：

- 隧道令牌不能写入仓库。
- 不要在聊天记录、文档或日志里重复粘贴令牌。
- 如果令牌已经泄露，应在 Cloudflare 后台轮换。
- 可以额外启用 Cloudflare Access，只允许指定邮箱访问。

## 数据和审计

- SQLite 数据库默认在 `data/app.db`。
- 数据库文件不应提交进仓库。
- 审计日志默认只记录操作元数据，不记录完整终端输入内容。
- 需要记录的典型事件包括登录、登出、项目创建、权限变更、会话启停、终端连接和终端输入元数据。

## 当前风险清单

- 管理员密码必须在部署时单独设置，不能写入仓库。
- 之前为了测试暴露过裸 `ttyd` 链路，最终必须确认 `terminal.example.com` 指向 `127.0.0.1:3020`。
- 如果 Cloudflare 隧道令牌曾经被明文发送过，建议项目完成后在 Cloudflare 后台轮换一次。
